Step 3. First-pass chromosome-by-chromosome `modkit extract full`

This is the recommended starting point for `merged_e5b`.
Do not start with one whole-genome `extract full` job.

Save as for example:
- `/data/leuven/383/vsc38330/thesis_dimelo/notebooks/bam_dual_thresholds/run_merged_e5b_extract_full_chr.sbatch`

```bash
#!/bin/bash
#SBATCH --job-name=merged_e5b_full_chr
#SBATCH -M genius
#SBATCH -A intro_vsc38330
#SBATCH --partition=batch
#SBATCH --time=12:00:00
#SBATCH --nodes=1
#SBATCH --ntasks=1
#SBATCH --cpus-per-task=36
#SBATCH --mem=180000M
#SBATCH --array=1-23%2
#SBATCH --output=/staging/leuven/stg_00118/BAM_Noemie/modkit_preproc/merged_e5b/slurm-%A_%a.out
#SBATCH --error=/staging/leuven/stg_00118/BAM_Noemie/modkit_preproc/merged_e5b/slurm-%A_%a.err

source /data/leuven/383/vsc38330/.venv/bin/activate

BAM=/staging/leuven/stg_00118/ONT_BAM_Noemie/bam_files/merged_e5b.sorted.bam

  "$BAM" \

  --region "$REGION" \
  --ref "$REF"
```

Step 4. Chunk-rescue strategy for chromosomes that OOM

If some chromosomes fail at the chromosome stage, rerun only those chromosomes as chunks.

Recommended first rescue chunk size:
- 50 Mb

If any 50 Mb chunk still OOMs:
- split that failed chunk into 25 Mb pieces

Chunk rescue SLURM template:
```bash
#!/bin/bash
#SBATCH --job-name=merged_e5b_full_chunk
#SBATCH -M genius
#SBATCH -A intro_vsc38330
#SBATCH --partition=batch
#SBATCH --time=12:00:00
#SBATCH --nodes=1
#SBATCH --ntasks=1
#SBATCH --cpus-per-task=36
#SBATCH --mem=180000M
#SBATCH --array=1-10%2
#SBATCH --output=/staging/leuven/stg_00118/BAM_Noemie/modkit_preproc/merged_e5b/slurm-retry-%A_%a.out
#SBATCH --error=/staging/leuven/stg_00118/BAM_Noemie/modkit_preproc/merged_e5b/slurm-retry-%A_%a.err

source /data/leuven/383/vsc38330/.venv/bin/activate
export PATH="/data/leuven/383/vsc38330/bin/dist_modkit_v0.6.1_481e3c9:$PATH"

BAM=/staging/leuven/stg_00118/ONT_BAM_Noemie/bam_files/merged_e5b.sorted.bam
REF=/data/leuven/383/vsc38330/thesis_dimelo/src/data/hg38.fa
INTERVALS=/staging/leuven/stg_00118/BAM_Noemie/modkit_preproc/merged_e5b/intervals_retry.tsv
OUTDIR=/staging/leuven/stg_00118/BAM_Noemie/modkit_preproc/merged_e5b/by_chunk

REGION=$(sed -n "${SLURM_ARRAY_TASK_ID}p" "$INTERVALS")
SAFE_REGION=$(echo "$REGION" | tr ':-' '__')
mkdir -p "$OUTDIR"

/data/leuven/383/vsc38330/bin/dist_modkit_v0.6.1_481e3c9/modkit extract full \
  "$BAM" \
  "$OUTDIR/extract_full_${SAFE_REGION}.tsv.gz" \
  --bgzf \
  --region "$REGION" \
  --ref "$REF"
```

Important assembly rule
- If a chromosome failed at the chromosome level and was later recovered by chunks, trust only the chunk outputs for that chromosome.
- Keep successful chromosome-level outputs for chromosomes that never OOMed.


Interpretation
- Keep `canonical_base = A` rows from `extract_calls_all.tsv.gz` for A-associated read-level analyses.
- Keep `canonical_base = C` rows from `extract_calls_cpg.tsv.gz` for CpG-only C read-level analyses.

Recommended operational order for `merged_e5b`
1. create and link the BAM index
2. run chromosome-by-chromosome `modkit extract full`
3. inspect which chromosomes, if any, OOMed
4. rerun only failed chromosomes as chunks
5. after extraction is stable, run `modkit pileup` for:
   - all A using `A:a`
   - combined CpG-only C using `C --combine-mods --cpg`

Short practical recommendation
- For `extract full`, start chromosome-by-chromosome rather than whole-genome.
- For `pileup`, use the `merged_c1`-style reference-aware commands above.
- Use `hg38.fa` for `merged_e5b`.
