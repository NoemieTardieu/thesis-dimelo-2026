8) PHASE 2 WINDOWED O/E ENRICHMENT HEATMAPS (BIN FEATURES, NOT PER-READ)
-----------------------------------------------------------------
Goal
- Reproduce Phase 1-style observed/expected Reg-vs-M enrichment heatmaps, but on genomic sub-bins inside each 1 Mb window.

New script
- /data/leuven/383/vsc38330/thesis_dimelo/src/preprocessing/phase2_long_context/windowed_oe_enrichment_heatmap.py

Feature definitions in this script
- Reg_bin:
  - default: A_mod_count per kb (`--reg-mode a_mod_per_kb`)
  - optional: A_mod / A_covered (`--reg-mode a_mod_frac`)
- M_bin:
  - default: C_methylated / C_covered (`--m-mode c_meth_frac`)
  - optional: C_mod_count per kb (`--m-mode c_mod_per_kb`)

Robustness stratification
- Coverage filter: keep only bins with `C_coverage >= X` via `--min-bin-c-coverage X`.
- Optional peaks stratification: pass mark-specific peak BEDs with `--peaks-bed-map mark=/path/to/file.bed`.

Resume behavior
- With `--resume`, the script writes per-mark feature caches in `--out-dir` and reuses them on rerun:
  `feature_cache_<mark>_bin<...>_<reg_mode>_<m_mode>_cov<...>.npz`
- With `--checkpoint-every 25`, partial checkpoints are saved during long runs:
  `feature_checkpoint_<mark>_bin<...>_<reg_mode>_<m_mode>_cov<...>.npz`
- Safe rerun pattern after interruption: run the exact same command again with `--resume`.

Flag support note
- `--resume` and `--progress-every` are supported for Step 5/8 build scripts.
- `--checkpoint-every` is supported by `windowed_oe_enrichment_heatmap.py` (Step 8), not by Step 5 backend builder.
- Step 6 (`summarize_backend_manifests.py`) is a short summarization command and does not expose resume/progress flags.

13) PHASE 3C - FINE-TUNING PLAN (NOT FULL PRETRAIN FROM SCRATCH)
-----------------------------------------------------------------
Step order
1. Load pretrained HyenaDNA checkpoint.
2. Freeze full backbone first; train only new embedding branch + prediction head(s).
3. Unfreeze top N backbone layers (partial unfreeze) after stable convergence.
4. Compare:
   - Baseline: D only
   - D + C methylation
   - D + C + Reg (optional)

Loss suggestions
- Main supervised task:
  predict Reg signal (binary or continuous depending on target definition).
- Optional multitask:
  loss_total = loss_reg + lambda_m * loss_methyl
- For methyl labels, always mask unknown positions (value 2).

Long-context constraints
- Start with short context smoke tests (1 kb to 10 kb).
- Scale to 100 kb, then 1 Mb after memory/performance validation.
- Keep gradient accumulation/checkpointing settings explicit in config.

Sanity checks before long runs
- Batch shape check:
  input_ids.shape == methyl_ids.shape == padding_mask.shape
- Unknown masking check:
  methyl loss excludes all positions with methyl_ids == 2
- Resume/reproducibility:
  save config + checkpoint + data index snapshot for each run
