========================================================

Purpose
- Test whether H3K27me3 and H3K4me3 are segregated in the Phase 2 data.
- Build three figures that directly address mark exclusivity and promoter/TSS organization.
- Test what can already be supported about CpG methylation antagonism from the existing Phase 2 tracks.
- State clearly what still requires an external CpG-island / CpG-density annotation.

Important interpretation note
- In this project, the histone-mark-specific signal track is the assay-derived A-mod / regulatory proxy:
  - `*.reg_a_mod_per_kb.bw`
- The CpG-methylation track is:
  - `*.meth_c_meth_frac.bw`
- So:
  - `reg_a_mod_per_kb` = histone-mark-associated A-mod signal
  - `meth_c_meth_frac` = CpG methylation fraction

1) FIGURE 1: GLOBAL H3K27me3 VS H3K4me3 JOINT-DENSITY
------------------------------------------------------------
Biological question
- Are H3K27me3 and H3K4me3 globally segregated across genomic bins?

Why this figure
- This is the clearest first test of mutual exclusivity.
- If the marks are strongly segregated, the density should favor:
  - low/low bins
  - high-H3K4me3 / low-H3K27me3 bins
  - high-H3K27me3 / low-H3K4me3 bins
- It should disfavor co-high bins.

Command
/data/leuven/383/vsc38330/.venv/bin/python /data/leuven/383/vsc38330/thesis_dimelo/src/preprocessing/genomic_context/plot_phase2_mark_relationships.py joint-density \
  --cache-a /data/leuven/383/vsc38330/thesis_dimelo/src/preprocessing/phase2_long_context/windowed_enrichment_1kb_combined/feature_cache_h3k4me3_bin1000_a_mod_per_kb_c_meth_frac_cov30.npz \
  --cache-b /data/leuven/383/vsc38330/thesis_dimelo/src/preprocessing/phase2_long_context/windowed_enrichment_1kb_combined/feature_cache_h3k27me3_bin1000_a_mod_per_kb_c_meth_frac_cov30.npz \
  --label-a H3K4me3_Amod_per_kb \
  --label-b H3K27me3_Amod_per_kb \
  --use-cov-intersection \
  --sample-size 500000 \
  --out-png /data/leuven/383/vsc38330/thesis_dimelo/src/preprocessing/genomic_context/h3k4me3_vs_h3k27me3_joint_density.png \
  --out-tsv /data/leuven/383/vsc38330/thesis_dimelo/src/preprocessing/genomic_context/h3k4me3_vs_h3k27me3_joint_density_stats.tsv

Expected outputs
- /data/leuven/383/vsc38330/thesis_dimelo/src/preprocessing/genomic_context/h3k4me3_vs_h3k27me3_joint_density.png
- /data/leuven/383/vsc38330/thesis_dimelo/src/preprocessing/genomic_context/h3k4me3_vs_h3k27me3_joint_density_stats.tsv

Interpretation
- Strong negative Pearson/Spearman or a clear depletion of co-high bins supports segregation.
- This is not proof of absolute mutual exclusivity, but it is strong global evidence.

2) FIGURE 2: TSS-CENTERED PROFILE SPLIT BY GENE CLASS
------------------------------------------------------------
Biological question
- Do active-like genes show high H3K4me3 and H3K27ac, while inactive/repressed genes show more H3K27me3?

Why this figure
- This is the most biologically interpretable figure.
- It ties the mark behavior to promoter/TSS biology rather than only genome-wide bins.

Requirement
- This figure needs an expression table.
- If no expression table is available locally, skip this step for now and go directly to Figure 3.

Step 2A: split the TSS BED by expression
/data/leuven/383/vsc38330/.venv/bin/python /data/leuven/383/vsc38330/thesis_dimelo/src/preprocessing/genomic_context/split_tss_by_expression.py \

  --expression-tsv /path/to/expression_table.tsv \
  --id-column gene_id \
  --value-column TPM \
  --bed-id-mode auto \
  --mode binary \
  --expressed-threshold 1.0 \
  --out-prefix /lustre1/scratch/383/vsc38330/tss_by_expression

- /lustre1/scratch/383/vsc38330/tss_by_expression.expressed.bed

ADDENDUM: 200 BP GROUPED PROFILES FOR CGI CLASSES AND ACTIVE/INACTIVE TSSs
==========================================================================

Why this addendum
- Several earlier profile figures used 1 kb signal tracks and were only displayed at finer bin sizes.
- If the goal is to make biologically cleaner grouped comparisons, the underlying signal resolution should match the plotting resolution more closely.
- This section therefore uses:
  - 200 bp source tracks from `signal_tracks_200bp`
  - 200 bp `computeMatrix` binning
- This is the correct setup when you want real 200 bp underlying information, rather than 1 kb tracks displayed at 50 bp.

Concept
- To put multiple biological groups on the same graph, use:
  - one signal track with `-S`
  - multiple BED files with `-R`
  - `plotProfile --perGroup`
- Then the different lines represent region classes, not different marks.

Required inputs
- 200 bp bigWigs:
  - /data/leuven/383/vsc38330/thesis_dimelo/src/preprocessing/genomic_context/signal_tracks_200bp/h3k27ac.reg_a_mod_per_kb.bw
  - /data/leuven/383/vsc38330/thesis_dimelo/src/preprocessing/genomic_context/signal_tracks_200bp/h3k27ac.meth_c_meth_frac.bw
  - /data/leuven/383/vsc38330/thesis_dimelo/src/preprocessing/genomic_context/signal_tracks_200bp/h3k27me3.reg_a_mod_per_kb.bw
  - /data/leuven/383/vsc38330/thesis_dimelo/src/preprocessing/genomic_context/signal_tracks_200bp/h3k27me3.meth_c_meth_frac.bw
  - /data/leuven/383/vsc38330/thesis_dimelo/src/preprocessing/genomic_context/signal_tracks_200bp/h3k4me3.reg_a_mod_per_kb.bw
  - /data/leuven/383/vsc38330/thesis_dimelo/src/preprocessing/genomic_context/signal_tracks_200bp/h3k4me3.meth_c_meth_frac.bw
- CGI promoter BEDs:

B1) HOW TO CREATE ACTIVE VS INACTIVE TSS BED FILES
------------------------------------------------------------
Why
- Claims about active transcription or repression require a real expression-based split.
- Without expression data, you cannot justify `active` vs `inactive` labels.

Input needed
- An expression table with at least:
  - one identifier column matching the TSS BED ids
  - one numeric expression column (for example TPM)

# stop here and continue
Command
/data/leuven/383/vsc38330/.venv/bin/python \
  /data/leuven/383/vsc38330/thesis_dimelo/src/preprocessing/genomic_context/split_tss_by_expression.py \
  --tss-bed /lustre1/scratch/383/vsc38330/TSS.bed \
  --expression-tsv /path/to/expression_table.tsv \
  --id-column gene_id \
  --value-column TPM \
  --bed-id-mode auto \

  --expressed-threshold 1.0 \
  --out-prefix /lustre1/scratch/383/vsc38330/tss

Interpretation
- These BEDs allow the same mark to be profiled over two transcriptional classes in the same graph.
- Use them only if the expression table is real and the id matching is validated.
